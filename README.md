# Локальная кольцевая «почта» (вариант 18)

Программа передачи сообщений между тремя узлами в кольцевой топологии с реализацией **физического**, **канального** и **прикладного** уровней (курс «Сетевые технологии в АСОИУ», МГТУ им. Баумана).

---

## Запуск на Windows

### 1. Установка Python

1. Скачайте **Python 3.10+** с [python.org](https://www.python.org/downloads/windows/).
2. При установке отметьте **«Add python.exe to PATH»** и **«tcl/tk and IDLE»** (нужен для графического интерфейса).
3. Проверьте в **cmd** или **PowerShell**:

```cmd
python --version
```

### 2. Подготовка проекта

Откройте терминал в папке проекта (например, `C:\Users\Имя\Documents\Курсач`):

```cmd
cd C:\Users\Имя\Documents\Курсач
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

После активации в начале строки появится `(.venv)`.

### 3. Демонстрация на одном ПК (рекомендуется)

Один запуск — три окна почты, эмуляция кольца RS-232 без кабелей:

```cmd
python demo.py
```

1. Дождитесь статуса **«Подключено»** во всех трёх окнах.
2. **Почта → Написать письмо** — выберите адресата (узел 2 или 3).
3. У получателя — двойной щелчок по письму → **Отметить прочитанным**.
4. У отправителя во **Исходящих** статус сменится на **«прочитано»**.

### 4. Три отдельных окна (три терминала)

В трёх окнах **cmd**, в каждом активируйте venv и запустите:

```cmd
python main.py --node 1 --demo-ring
python main.py --node 2 --demo-ring
python main.py --node 3 --demo-ring
```

Флаг `--demo-ring` подключает общий программный симулятор кольца (как в `demo.py`).

### 5. Реальные COM-порты (com0com, по ТЗ)

Для защиты с «настоящими» портами:

1. Установите [com0com](https://sourceforge.net/projects/com0com/) (Signed Driver).
2. Создайте **три пары** виртуальных портов и соедините их в кольцо (см. схему в разделе «Физический уровень»).
3. Запустите три экземпляра:

```cmd
python main.py --node 1 --serial-left COM11 --serial-right COM12
python main.py --node 2 --serial-left COM21 --serial-right COM22
python main.py --node 3 --serial-left COM31 --serial-right COM32
```

Имена `COM*` замените на свои из настройки com0com. Параметры скорости задаются в меню **Сеть → Параметры COM** (по умолчанию 115200 8N1).

### 6. Типичные проблемы на Windows

| Проблема | Решение |
|----------|---------|
| `python` не найден | Переустановите Python с галочкой PATH или используйте `py -3 demo.py` |
| Нет окна / ошибка tkinter | Переустановите Python с компонентом **tcl/tk** |
| Порт занят | Закройте другие программы, использующие COM |
| Кириллица в пути | Скопируйте проект в путь без проблемной кодировки, например `C:\kursovaya` |

## Сценарий демонстрации

1. В окне **узла 1**: **Почта → Написать письмо** → адресат **узел 2** → отправить.
2. В окне **узла 2**: двойной щелчок во **Входящих** → **Отметить прочитанным**.
3. В окне **узла 1**: во **Исходящих** — «отправлено» → «доставлено» → «прочитано».
4. Внизу каждого окна — **протокол событий** (передача, ACK, ошибки).

## Как работает система: обзор

Три персональных компьютера соединены в **ненаправленное кольцо** (у каждого два соседа). Сообщения идут **по кольцу** от узла к узлу; промежуточные узлы **ретранслируют** кадры, если адресат не они.

```
        ┌─────────┐
        │ Узел 1  │
        └──┬───┬──┘
    left │   │ right
           │   │
    ┌──────┘   └──────┐
    │                 │
┌───┴───┐         ┌───┴───┐
│Узел 3 │         │Узел 2 │
└───┬───┘         └───┬───┘
    │                 │
    └────────┬────────┘
             │
      (замкнутое кольцо)
```

Данные проходят стек **сверху вниз** при отправке и **снизу вверх** при приёме:

```
[ GUI ]  ←→  Прикладной  ←→  Канальный  ←→  Физический  ←→  линия RS-232
```

## Физический уровень (`src/physical/`)

### Назначение

Имитирует работу интерфейса **RS-232C** и **нуль-модемного** соединения с соседями: открытие порта, настройка скорости, побайтовая передача и приём, поддержание канала.

### Состав

| Файл | Роль |
|------|------|
| `transport.py` | Класс `PhysicalLayer` — два порта (`left`, `right`), потоки чтения |
| `ring_simulator.py` | `RingSimulator` + `VirtualPort` — эмуляция кольца на одном ПК |

### Два порта на узле

Каждый ПК в кольце имеет **два соседа**:

- **`left`** — связь с предыдущим узлом в кольце (3→1, 1→2, 2→3);
- **`right`** — связь со следующим (1→2, 2→3, 3→1).

При **отправке** канальный уровень выбирает порт (`left` или `right`) в сторону адресата. При **приёме** физический уровень передаёт байты наверх вместе с меткой стороны (`"left"` / `"right"`), чтобы канальный уровень знал, в какой порт **ретранслировать** чужой кадр.

### Режимы работы

1. **Демо (`demo.py`, `--demo-ring`)**  
   `RingSimulator` связывает очереди TX/RX трёх узлов в кольцо:
   - `right(1) → left(2) → right(2) → left(3) → right(3) → left(1)` (по часовой);
   - `left(N) → right(N-1)` (против часовой).  
   Задержка передачи пропорциональна скорости (имитация времени на линии).

2. **Реальные COM (`pyserial`)**  
   `PhysicalLayer` открывает два последовательных порта Windows (`COM3`, `COM4`, …) с параметрами 8N1. Настройка — меню **Сеть → Параметры COM** (скорость 9600–921600 бод).

### Установка и разрыв канала

- **Сеть → Подключить канал** — `open()` на обоих портах, запуск потоков `_read_loop_*`.
- **Отключить** — закрытие портов, остановка потоков.
- Статус отображается в панели инструментов.

### Что передаётся

Только **сырые байты** — без понятия «письмо» или «кадр». Сборка кадров — задача канального уровня (`FrameDecoder` буферизует поток).

## Канальный уровень (`src/datalink/`)

### Назначение

Обеспечивает **логическую** передачу между узлами: кадрирование, **CRC-16-CCITT**, маршрутизация по кольцу, служебные кадры подтверждения и ошибок.

### Формат кадра

```
┌──────┬─────┬─────┬──────┬─────┬─────┬─────┬──────────┬────────┬──────┐
│ FLAG │ DST │ SRC │ TYPE │ HOP │ SEQ │ LEN │ PAYLOAD  │ CRC16  │ FLAG │
│ 0x7E │  1  │  1  │  1   │  1  │  2  │  2  │ 0..4096  │   2    │ 0x7E │
└──────┴─────┴─────┴──────┴─────┴─────┴─────┴──────────┴────────┴──────┘
```

| Поле | Описание |
|------|----------|
| **DST** | Адрес получателя (1, 2 или 3) |
| **SRC** | Адрес отправителя |
| **TYPE** | Тип кадра (см. ниже) |
| **HOP** | Счётчик переходов по кольцу (макс. 3 — защита от зацикливания) |
| **SEQ** | Номер кадра для отсечения дубликатов |
| **PAYLOAD** | Полезные данные (JSON письма или ACK) |
| **CRC16** | Контрольная сумма CCITT (полином 0x1021) по заголовку + payload |

Между флагами применяется **byte stuffing** (как в HDLC): байты `0x7E` и `0x7D` в теле кадра экранируются.

### Типы кадров (`src/common/constants.py`)

| Код | Имя | Назначение |
|-----|-----|------------|
| `0x01` | `FRAME_DATA` | Письмо (прикладной payload) |
| `0x02` | `FRAME_ACK_DELIVERED` | Подтверждение доставки |
| `0x03` | `FRAME_ACK_READ` | Подтверждение прочтения |
| `0x04` | `FRAME_ERROR` | Служебное сообщение об ошибке соседу |

### Отправка письма (узел 1 → узел 2)

1. Прикладной уровень формирует JSON письма.
2. `DataLinkLayer.send_letter()` упаковывает его в кадр `DATA`, `hop=0`, вычисляет CRC.
3. Выбирается направление: **кратчайший путь** по кольцу (`right` или `left`).
4. Кадр кодируется (`encode_frame`) и уходит в `PhysicalLayer.send()`.

### Ретрансляция (узел 3 пересылает кадр 1→2)

1. Кадр приходит на порт `left` или `right`.
2. `FrameDecoder` собирает кадр из потока байтов.
3. Если `DST ≠ мой узел` и `HOP < 3` — кадр пересылается на **противоположный** порт, `HOP` увеличивается на 1.
4. Если `DST = мой узел` — кадр обрабатывается локально, **не** ретранслируется.

### Защита от дубликатов и ошибок

- Пара `(SRC, SEQ)` запоминается на ~30 с; повторный кадр **отбрасывается** (защита от обхода кольца).
- При неверном CRC кадр **не разбирается** (в журнале можно увидеть отсутствие доставки).
- `FRAME_ERROR` информирует соседний узел о проблеме (по ТЗ).

### Подтверждения

| Событие | Кадр | Когда |
|---------|------|--------|
| Письмо принято получателем | `ACK_DELIVERED` | Сразу после успешного разбора `DATA` |
| Пользователь открыл письмо | `ACK_READ` | После нажатия «Прочитано» в GUI |

ACK маршрутизируются по кольцу так же, как `DATA`, до узла `SRC` исходного письма.

## Прикладной уровень (`src/application/`)

### Назначение

Реализует услугу **«почта»**: письма, адресация между тремя абонентами, статусы, уведомление отправителя о прочтении.

### Модель данных (`mail.py`)

**Письмо (`Letter`):**

| Поле | Описание |
|------|----------|
| `letter_id` | Уникальный идентификатор (8 символов UUID) |
| `sender` | Узел-отправитель (1..3) |
| `recipient` | Узел-получатель |
| `subject`, `body` | Тема и текст |
| `status` | `draft` → `sent` → `delivered` → `read` |

**Подтверждение (`AckPayload`):** `letter_id` + тип (`delivered` / `read`).

Сериализация — **JSON в UTF-8** внутри payload кадра.

### Логика (`app_layer.py`)

**Исходящие:**

1. Пользователь создаёт письмо → статус `sent`, запись в `outbox`.
2. Вызов `datalink.send_letter()`.
3. При `ACK_DELIVERED` → статус `delivered`.
4. При `ACK_READ` → статус `read` (отправитель видит «прочитано»).

**Входящие:**

1. Приём `DATA` → письмо в `inbox`, статус `delivered`.
2. Канальный уровень автоматически шлёт `ACK_DELIVERED` отправителю.
3. Пользователь открывает письмо → `open_letter()` → `ACK_READ` отправителю.

## Графический интерфейс (`src/ui/`)

Не является сетевым уровнем по ISO, но по ТЗ входит в **прикладной** уровень как «интерфейс с пользователем».

| Элемент | Функция |
|---------|---------|
| **Входящие** | Список полученных писем; двойной щелчок — просмотр и «прочитано» |
| **Исходящие** | Список отправленных; обновление статуса по ACK |
| **Протокол событий** | Лог сети: отправка, приём, ACK, ошибки |
| **Меню «Сеть»** | Подключение, параметры COM |
| **Меню «Почта»** | Создание письма, выбор адресата (1/2/3, кроме себя) |

Обновление GUI из фоновых потоков приёма — через `root.after(0, ...)` (потокобезопасность tkinter).

## Сборка узла (`src/node.py`)

Класс `NetworkNode` связывает уровни:

```
NetworkNode
├── PhysicalLayer   ← байты с линии
├── DataLinkLayer   ← кадры, маршрутизация
└── ApplicationLayer ← письма, статусы
         ↑
    NodeWindow (GUI)
```

Точка входа демо: `demo.py` — один процесс, общий `RingSimulator`, три окна `NodeWindow`.
